Electron insert factor modelling

Description

This online web app is for the modelling of the portion of the electron output factor that is dependent on the shape of the shielding insert mounted within the applicator. This allows modelling insert factors using only the measured factors already available at a centre. Should all outliers be removed from the data set the user might expect as low as 0.5% standard uncertainty for factor prediction with as little as 8 data points.

If you find bugs, or have any usability suggestions please email me so that I may make appropriate fixes and improvements.

Publication

The paper detailing the algorithim and its validation:

S. Biggs, M. Sobolewski, R. Murry, J. Kenny, Spline modelling electron insert factors using routine measurements. Physica Medica (2015), doi:10.1016/j.ejmp.2015.11.002.

Web app structure

This web app is divided into six logical components all of which can be accessed from the navigation menu. These are outlined below:

  • Machine specifications component
    • Used to define your centre specific machine specifications.
  • Dicom component
    • Used to extract patient insert shapes from Dicom files.
  • Parameterisation component
    • Used to parameterise insert shapes into equivalent ellipse widths and lengths for use within the empirical model.
  • Create model component
    • Use this to create the model once there are at least 8 measured factors for a given machine, applicator, energy, ssd combination.
  • Use model component
    • Use this to quickly calculate small or large numbers of predicted factors.
    • Measured factors can be given here to batch evaluate the models ability to predict factors for validation purposes.
  • Database management component
    • Use this to transfer and backup the data within the local browser data.

Demo Instructions

If you wish to simply demo this app these are the following steps you can take:

  • Within the dicom component download the demo dicom file, load it into the web app and see the insert shapes be pulled from the dicom file without any network traffic
  • Within the parameterise component press the "Load Demo Data" button and then press the "Parameteterise Insert" button to see the parameterisation algorithim in action
  • Within the model component press the "Load Demo Data" button and then press the "Run Spline Model" to see the spline model being created. You can then either hover over the model, or tap on the model to view the relevant values.

Implementation instructions

To implement this web app in your centre you need to do the following:

  • Specify the machine models and the corresponding applicators, energies, and ssds within the machine specification component.
  • Collate / measure your electron insert factor measurements. These factors are to measured in accordance to TG 25. Measured insert factors from regular or irregular shapes are able to be used.
  • Parameterising new insert shapes
    • If the insert shapes are ellipses, you may directly input the ellipse widths and lengths as projected to the isocentre within the create model component.
    • If you know the x and y coordinates of your insert then determine the equivalent ellipse widths and lengths by inputting x and y coordinates as defined at isocentre within the parameterisation component.
    • If the insert shape is defined within your planning system, export the patient plan to dicom and then read the insert coordinates from the dicom file using the dicom component.
    • If you have a batch of generic insert shapes you can directly input the widths and areas of the shapes into the create model component. The width is defined as the diameter of the largest fully encompassed circle of your insert shapes at isocentre. The area is defined as the insert shape area projected to isocentre. Rectangles can be input in this way by simply inputting the rectangle's width projected to isocentre as width, and its area projected to isocentre as area.
  • Once you have the corresponding equivalent ellipse widths and lengths for your measured factors these may be appended to the relevant machine configuration within the create model component
  • Whenever a machine configuration gets at least 8 data points you may press the "Run Spline Model" on the create model component which will allow the prediction of insert factors for other widths and lengths.
  • The use model component is specifically designed to aid you in looking up factors and being able to validating your model by looking up large batches of data for which you already know the factor.
  • Once you are content with your model use the database management component to export and backup your data. This data can be placed on a shared drive for access by users who need to use the model.
  • If you need to have a consistent reliable interface you may wish to consider downloading a copy of this web app and serving it within your centre's intranet. This can be achieved by following the relevant steps given within the further details component.
